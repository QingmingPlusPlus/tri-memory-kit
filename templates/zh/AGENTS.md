# AGENTS 总纲（<project-name>）

本文件只保留仓库级最小硬约束。详细治理规则见 `doc/agent-guide.md`，开发、评审、排障或文档改动前的记忆入口见 `doc/agent-memory-map.md`。

## 仓库边界

- `<app-a>` 只承载平台 A 特有 UI 与平台适配。
- `<app-b>` 只承载平台 B 特有 UI 与平台适配。
- `<shared>` 只承载跨端复用业务逻辑、请求封装、类型定义与纯工具函数。
- 禁止端与端直接依赖；跨端复用统一经 `<shared>` 暴露稳定接口。

## 记忆路由

- 小范围局部改动，先读 `doc/agent-memory-map.md` 中的相关行。
- 功能开发、评审、排障或文档/spec/skill 改动，先按路由表读取匹配的 `doc`、`spec` 和 skill，再动手。
- 若路由表没有匹配项，先用 `rg` 在 `doc/`、`spec/`、`openspec/specs/`、`.agents/skills/` 中查找最近记忆；本次改动形成长期入口时补充路由表。

## 每次改动清单

1. 标记本次改动归属模块。
2. 判断是否可跨模块复用；可复用内容优先下沉到共享层。
3. 明确记忆落点：约束未来行为进 spec，描述当前项目进 `doc/`，指导重复操作进 `.agents/skills/`。
4. 若新增或调整模块入口、长期文档、spec 或 skill，补充 `doc/agent-memory-map.md`。
5. 自检边界：确认没有引入错误层级依赖。

## 冲突优先级

当记录互相冲突时，优先级为：用户当前指令 > spec > tests > 当前实现 > doc > skill/历史说明。

## 安全

不要把密钥、凭据、客户隐私数据或短期排障记录写入项目记忆。
